chapter 1. 머신러닝

머신러닝이란 무엇인가

  1. 위키에서의 정의
    • 머신러닝은 인공지능의 패턴인식과 계산학습 이론에서 발전한 컴퓨터 과학의 한 분야다. 머신러닝은 데이터로부터 학습하고, 예측할 수 있는 알고리즘을 연구한다. 이러한 알고리즘은 정적으로 주어진 프로그램이 아닌 입력된 데이터로부터 모델을 만들어 예측이나 결정을 내린다.
  2. 기존 접근방법과 머신러닝 접근 방법의 비교
    • 여기에서 프로그램을 작성한다는 것은 스스로 모델을 만들과 적절한 파라미터를 설정하는 것을 의미
  3. 머신러닝의 특성
    • 머신러닝에 있어 데이터는 매우 중요한 요소
    • garbage in, garbage out 을 잊지 말아야 함!!!
    • 데이터가 머신러닝의 결과물에 막대한 영향을 미치므로, 데이터를 탐색하고 정리하는 데이터 마이닝이 아주 중요함

머신러닝의 장단점

  1. 머신러닝의 장점
    • 학습을 위한 지식 표현이 필요 없다. 그러나 컴퓨터가 지식을 이해하게 하기 위한 표현은 어려운 문제다.
    • 충분한 데이터와 접합한 알고리즘을 사용한다면 사람이 만든 모델보다 좋은 결과를 보여줄 수 있다.
    • 고도의 수학적 지식이나 프로그래밍 능력을 요구하지 않는다. 기본적인 지식과 능력으로도 충분히 머신러닝을 이용할 수 있다.
    • 자동화가 가능하다. 프로그램으로 머신러닝을 학습시키고 최적의 파라미터를 찾아 그 결과에 대한 평가 등을 자동화하여 진행할 수 있다.
    • 저렴하고 유연하다. 데이터를 제외한 나머지 과정을 자동화가 가능하므로 저렴하게 실행할 수 있다.
    • 프로그램을 이용해 자신이 원하는 대로 사용할 수 있다.
  2. 머신러닝의 단점
    • 데이터 준비에 많은 노력이 든다. 지도학습의 경우 모든 개별 데이터에 결과치를 만들어 주어야 한다.
    • 오류 발생이 쉽다. 일반적으로 정확도가 높은 모델을 만들기 어렵다.
    • 생성된 모델이 블랙박스이기 때문에 이를 해석하기가 어렵다. 정확도를 높이려면 모델을 수정하거나 개선할 수 있어야 하는데, 대부분의 머신러닝 알고리즘은 학습결과로 생성된 모델을 이애하기 어렵고 모델 자체를 개선할 수 없다.
    • 과적합 문제가 종종 발생한다. 주어진 데이터에 너무 최적화되어 학습에 사용된 데이터에는 높은 예측력을 보여주지만, 그렇지 않은 데이터에는 좋지 않은 예측력을 보여준다.
  3. 머신러닝의 종류
    • 지도학습 : 사람이 입력과 출력을 모두 제공
      • 스팸메일 필터링, OCR 문자 인식 등
      • 일종의 최적화(optimization) 문제
      • 하나의 데이터는 입력과 출력이 같이 묶여있는 튜플 형태
    • 비지도학습 : 사람이 입력만 제공
      • 입력 데이터의 구조를 파악하거나 관계를 분석하는 방법
      • 지식발견(knowledge discovery)라고도 함
      • 학습의 결과로 생각하지 못한 지식을 반견하거나 입력 데이터 간의 그룹 또는 특성 등을 발견할 수 있음
      • 지도학습과 구분되는 특징은 학습결과에 대한 평가가 힘들다는 점 <- 학습 시 출력을 정해주는 것이 아니기 때문에 평가기준을 잡을 수가 없기 때문임
    • 강화학습 : 어떤 환경에서 특정 목표를 달성하기 위해 스스로 학습
    • 사용 빈도 : 지도학습 > 비지도학습 > 강화학습

머신러닝이 할 수 있는 것

  1. 머신러닝으로 할 수 있는 것
    • 머신러닝의 방법으로 해결할 수 있는 문제가 사실적으로 그렇게 많지는 않음
    • 하나의 완성된 결과를 얻기 위해서는 많은 시도를 통해서 조금씩 개선을 해야 하고, 때에 따라서는 모델을 새롭게 만들거나 전혀 새로운 시각으로 접근해야 한다.
    • 크게 3가지 종류의 일을 처리할 수 있음
      • 변수간의 관계를 파악하는 회귀
      • 데이터를 분류하는 분류
      • 데이터를 연관있는 것끼리 묶어주는 군집화
  2. 회귀
    • 연속적인 숫자 변수들간의 상관관계를 파악하는 것
    • 종속변수와 독립변수 사이의 연관성을 분석하는 것을 주된 목적으로 함
    • 회귀 분석 시 가장 먼저 해야 할 일은 데이터의 상관정도를 직관적으로 파악하게 종속변수와 독립변수의 산점도를 그리는 것
  3. 분류
    • 데이터를 분류에 따라 나누는 것
    • 회귀와는 다르게 어떤 값을 예측하는 것이 아니라 어떤 종류에 속하는지를 파악하는 문제
    • 회귀와 마찬가지로 산점도를 그래 데이터를 파악해야 한다.
    • 회귀는 연속적인 데이터에 적용할 수 있지만, 분류는 범주형 데이터에 적용할 수 있음
  4. 군집화
    • 데이터를 유사한 특성을 가진 무리(cluster)로 묶는 것을 의미함
    • 비지도학습에서 사용하는 것으로 출력 데이터 없이 입력 데이터만으로 이뤄지고, 주로 데이터의 특성을 파악하거나 이애하기 위해 적용 됨
    • 예를 들어, 마케팅 캠페인을 진행하는데 어떤 특성을 가진 사람들이 마케팅 캠페인에 반응하는지를 알고 싶다고 하면, 어떤 기준으로 대상자를 선정하는 것이 좋을지 모를 수 있음. 이럴때 군집화가 효과적으로 사용될 수 있음
    • 군집화는 주어진 데이터들의 유사도를 계산해서 비슷한 특성이 있다고 판단되는 것끼리 군집으로 분류하는 것
    • 군집으로 분류된 데이터의 유사한 특성을 파악

머신러닝 알고리즘

  • 알고리즘마다 특성이 달라서 원하는 것이 무엇인지를 명확하게 하고, 그 결과를 얻기 위해서는 어떤 종류의 문제에 속하는지를 판별한 다음 머신러닝에 적용할 알고리즘을 선택해야 함
  • 적합한 알고리즘을 선택하기 위해서는 데이터의 크기와 데이터 타입(문자, 숫자, 이미지 등), 데이터가 선형인지 비선형인지 등의 요소를 고려해야 함
  • 회귀
    • 선형 회귀
    • SGD(stochastic gradient descent) 회귀
    • 서포트 벡터 회귀
    • 랜덤 포레스트 회귀
    • 베이지안 회귀
    • 등위 회귀
    • 베이지안 ADR 회귀
  • 분류
    • 로지스틱 회귀
    • 서포트 벡터 머신
    • 랜덤 포레스트
    • 의사결정 트리
    • 그래디언트 부스팅 트리
    • SGD 분류기
    • AdaBoost
  • 군집화
    • K-means
    • 스펙트럼 군집화
    • 가우시안 혼합
    • 병합식 군집화
    • 친근도 전파
    • 평균 이동
  • 머신러닝 알고리즘을 사용하는 개발자는 머신러닝 알고리즘의 개념과 사용법, 알고리즘의 특성을 아는 것이 중요함
  • 머신러닝으로 해결하려는 문제에 효과가 있다고 알려진 알고리즘이 없다면, 적용할 수 있는 모든 머신러닝 알고리즘을 하나씩 적용해 시행착오를 거쳐 가장 좋은 성능을 보여주는 것을 채택해 사용하도록 함

머신러닝 프로세스

  1. 프로세스의 중요성
    • 한두번의 시도가 아니라 수십, 수백 번의 시행착오를 거쳐 모델을 완성하므로 프로세스가 결과물의 품질에 절대적인 영향을 미침
  2. 첫 번째 전처리(pre-processing)
    • 학습에 사용될 데이터인 학습용 데이터셋을 준비하는 과정
    • 데이터를 선별하고 빠진 데이터가 있는지를 확인하며, 데이터에 왜곡을 줄 수 있는 이상치 데이터를 처리
    • 필요에 따라서는 정규화 작업을 진행
  3. 학습용 데이터셋
    • 데이터를 만드는 과정으로, 지도학습이면 입력과 출력, 비지도 학습이면 입력 데이터를 정해진 형태로 생성
  4. 두 번째 전처리
    • 데이터셋에 포함된 변수 중 어떤 것을 학습에 사용할지를 결정(feature selection)
    • 학습에 지대한 영향을 미치는 중요한 과정
    • 데이터의 성격이나 적용하려는 알고리즘에 따라 feature scaling과 같이 데이터를 일정 범위로 변환하는 작업도 이루어짐
  5. 머신러닝 알고리즘 학습
    • 준비된 학습 데이터셋을 이용해서 학습을 해보는 과정
  6. 파라미터 최적화
    • 선택한 알고리즘에서 적용 가능한 파라미터 값들을 조정해 결과물의 품질을 높이는 과정
    • 사람이 조정한 파라미터 값에 따라 학습 품질을 파악해서 최적의 파라미터를 찾거나 격자 탐색(grid search)를 이용해 알고리즘이 스스로 최적의 값을 찾도록 함
  7. 후처리
    • 학습결과의 품질평가가 주된 목적
    • 하나의 알고리즘이나 모델만을 적용하는 것이 아니므로 다수의 모델과 알고리즘 중 어떤 것이 가장 좋은 결과를 가져오는지, 각 알고리즘이 주어진 문제에 어떻게 반응하는지를 평가
  8. 최종모델
    • 완성된 모델을 이용해 실제 데이터에 적용

No Free Lunch Theorem

  • 특정한 문제에 최적화된 알고리즘은 다른 문제에서는 그렇지 않다는 것을 수학적으로 증명한 정리
  • 학습용 데이터셋에 종속변수와 독립변수의 관계를 분석하고 이를 기반으로 모델을 만드는 것이므로 철저하게 학습용 데이터셋에 종속
  • 따라서 해결하려는 문제를 잘 정의하고, 연관된 데이터를 잘 정제하고, 머신러닝을 적용해 최적화 해야 함

chapter 1. 주식이 뭐예요?

주식이란 무엇인가?

  1. 주식회사
    • 주식회사는 한 사람이 만든 회사가 아니라 여러 사람이 돈을 투자해서 만든 회사
    • 여러 사람이 투자해서 만들었기 때문에 책임과 권한도 나눠 가짐
  2. 주식회사 설립 절차
    • 자본금 규모와 1주 당 금액을 신고해야 함
    • 자본금 규모 / 1주 당 금액 = 주식 발행 수

자본금과 주식의 상관관계 - 증자/감자

  • 주식회사의 자본금은 늘기도 하고 줄기도 함
  • 증자란
    • 기업이 주식을 추가로 발행하여 자본금을 늘리는 것
    • 주식을 돈을 받고 팔면서 증자를 하면 유상증자, 공짜로 나눠주면 무상증자라고 함
    • 공모주 청약을 통해 신규 주식을 모집하는 경우는 유상증자에 해당
  • 감자란
    • 기업의 규모를 축소하거나 합병할 때 자본금을 줄이는 것을 말함
    • 유상감자는 주주에게 현금을 돌려주고 주식수를 줄이는 것을 말하고, 무상감자는 자본잠식이 있을 때 기업이 재무구조를 건실화하기 위해 대가 없이 주식 수를 줄이는 것을 말함
    • 따라서 무상감자의 경우는 감자액 비율만큼 손실을 보게 됨

무상증자와 유상증자, 언제나 호재인가?

  1. 무상증자의 경우
    • 무상증자는 주주에게 주식을 공짜로 나누어주는 것이지만 기업재무 측면에서 보면, 기업의 유보금을 헐어 주식을 발행 한 뒤 주주에게 나눠주는 것이므로 기업의 전체 가치에는 변화가 없음
    • 따라서 기업의 전체 자본금은 그대로인 것이므로 1주당 가치는 감소
    • 이론적으로는 무상증자 비율만큼 주가가 하락해야 하지만, 현실적으로 무상증자 권리락은 그보다 적게 떨어지는 경향이 있음
    • 무상증자를 하면 주당 가격이 큰 폭으로 떨어지기 때문에 주가가 싸 보이는 착시 현상이 일어나서 매수세력이 몰리기 때문임
    • 그래서 장기적인 관점에서 보면 무상증자는 호재도 악재도 아님
    • 권리락 : 무상증자 이전에 주가를 낮추어 증자 이전 수준의 수익률과 비슷하게 만드는 것
  2. 유상증자의 경우
    • 증자 규모와 할인율에 따라 호재가 되지 못하고 악재로 작용하는 경우가 있음
    • 할인율은 낮은데 비해 증가 규모가 큰 경우는 증자 후 물량 부담으로 인해 주가가 하락할 수 있기 때문임
    • 유상증자는 기업 전망이 좋거나 증권시장 전망이 좋을 때는 상당한 호재이므로 청약하는 것이 좋음
    • 할인율 : 주주의 청약을 유도하기 위해 10 ~ 30%의 할인율을 적용해 시장가격보다 낮게 신주를 발행. 유상증자의 경우는 할인율이 높을수록 유리

주식은 사고팔 수 있는 것 - 유통시장, 발행시장

  1. 유통시장
    • 대형 우량주를 주로 거래하는 '코스피 시장'과 벤처기업 주식을 거래하는 '코스닥'과 같이 이미 발행된 주식을 거래하는 시장
  2. 발행시장
    • 신규 상장이나 공모주 청약을 통해 자금 수요자인 기업이 주식을 새로 발행하여 자금 공급자인 투자자들에게 최초로 판매하는 시장
  3. 기타
    • 증권시장 = 유통시장 + 발행시장
    • 공모주 청약은 언제나 이익이 되는가?
      • 공모주 청약은 시장가격보다 낮게 책정되는 경향이 있어서 이익이 되는 경우가 많음
      • 다만 동일 업종의 주가와 상대평가를 하기 때문에 증권시장이 과열되어 있을 때는 공모가가 높게 책정되는 경우도 있기 때문에 조심
      • 기업의 성장성을 보고 투자하는 것이라면 상장 이후 주가가 청약가보다 당장 높지 않더라도 장기적으로 보면 더 유리할 수 있기 때문에 수익을 낼 수 있는 투자방법임

chapter 2. 통계란

통계의 기본 개념과 용어

  1. 모집단과 표본
    • 모집단 : 관심있는 문제와 연관된 전체 집합
    • 표본 : 전체 데이터로부터 추출된 일부 집합
  2. 파라미터와 통계량
    • 파라미터 : 모집단으로부터 얻은 데이터의 성격을 나타내주는 지표(모평균, 모분산 등)
    • 통계량 : 표본으로부터 얻은 데이터의 성격을 나타내주는 지표(표본 평균, 표본 분산 등)
  3. 표집오차(sampling error)
    • 모집단과 표본의 차이로 발생하는 오차
    • 아무리 많은 노력을 기울여 지능적인 표집 방법을 사용한다 하더라도 표본과 모집단의 특성이 100% 일치할 수는 없음
    • 모집단이 아닌 표본을 사용한다면 언제나 표집오차가 존재하므로 이를 염두에 두고 데이터를 사용해야 함
    • 결국 머신러닝에 사용하는 모든 학습용 데이터셋이 과연 얼마나 모집단을 잘 반영하는지에 대해 검증해 보아야 한다는 말
  4. 종속변수와 독립변수
    • 종속변수
      • 어떤 목적에 대한 결과를 파악하기 위해 사용하는 변수
      • 반응변수(response), 측정변수(measured), 피예측변수(predicted), 피설명변수(explained), 출력변수(output) 등으로 불림
    • 독립변수
      • 예측변수(predictor), 특징(feature), 입력변수(input), 설명변수(explanatory) 등으로 불림
  5. 연속변수와 이산변수
    • 통계처리를 하려는 값의 특성에 따라 분류한 것
    • 연속변수
      • 실수로 표현할 수 있는 변수
      • 특정 범위에서 허용되는 값이 '셀 수 없다'
    • 이산변수
      • '셀 수 있는' 변수
    • 이를 구분해야 하는 이유는 이에 따라 적절한 방법을 선택해야 하기 때문
  6. 모델
    • 대상을 설명하기 위한 표현 방법

chapter 3. 시계열 데이터

  • 알고리즘 트레이딩을 위해서는 주가 데이터의 움직임을 잘 표현할 수 있는 모델이 필요
  • 주가의 움직임에 영향을 미칠 수 있는 요소(feature)를 포함하고 있어야 하고, 이게 얼마나 잘 만들어졌냐에 따라 주가 예측의 정확도가 결정
  • 주가와 같은 금융 시계열 데이터를 다룰 때는 머신러닝을 사용하더라도 전적으로 여기에 의존하기 보다는 일정 수준 이상의 수학적 모델을 반디시 같이 활용해야 함
  • 확보가능한 데이터가 무엇인가를 파악
  • 데이터 내의 변수 간의 상관관계나 시간에 따른 변화를 파악
  • 자기상관(autocorrelation)이나 자기공분산(autocovariance) 같은 방법을 사용하면 시간에 따라사 값이 어떻게 영향을 미치고, 그에 따른 결과가 어떻게 변화하는지를 알아내는데 많은 도움이 됨

시계열 데이터

  • 시계열 데이터
    • 일정 시간 간격으로 측정된 순서를 갖는 데이터를 일컬음
    • 시간의 흐름에 따라 변수가 어떻게 변화하는지를 보여주는 것으로 데이터에 순서가 있고, 그 값이 지속적으로 변화함
    • 순서를 뒤집거나 무작위로 데이터를 추출해 사용할 수 없음
    • 시간에 따른 종속변수의 변화량을 측정해야 해서 일정한 간격으로 측정해야 한다는 점도 중요
    • 수집 시에는 반드시 동일한 조건과 방법으로 측정해 관측한 데이터가 객관성을 가질 수 있도록 해야 함

시계열 데이터 분석

  1. 시계열 데이터 분석이란
    • 시계열 데이터의 패턴을 파악하거나 패턴에 큰 영향을 미치는 요소를 찾는 작업을 시계열 데이터 분석이라고 함
  2. 분석 목적
    • 시계열 데이터 패턴에 영향을 미치는 요소(feature)를 찾음
    • 과거의 데이터가 어떻게 미래의 데이터에 영향을 미치는지를 분석
    • 미래의 데이터를 예측

주요 시계열 데이터의 특성

  1. 주요 특성
    • trend : 측정값이 시간의 흐름에 따라 증가나 감소 또는 반복 등의 일정한 패턴이나 경향을 가지고 있는가
    • seasonality : 일/월/년/계절 등 일정 시간에 따라 지속적으로 반복되는 패턴이 있는가
    • outlier : 다른 값들과 동떨어진 이상치를 관측할 수 있는가
    • long-run cycle : 계절성과는 별도로 오랜 기간 반복되는 패턴이 있는가
    • constant variance : 측정값이 일정한 수준 이내로 변동되는가 아니면 변동이 무작위로 발생하는가
    • abrupt change : 급격한 변동을 보이는 데이터가 있는가
  2. 그 외 중요
    • 특성을 파악할 때 데이터의 범위 또한 매우 중요함
    • 동일한 데이터 이더라도 범위에 따라 일정 패턴이 보일수도 있고 그렇지 않을수도 있기 때문

랜덤과정(stochastic process)

  1. 랜덤과정이란
    • 확률변수가 시간의 흐름에 따라 변화하는 값들
    • 시간의 흐름에 따른 확률 분포로 생각하면 됨
  2. 결정적 과정이란
    • 랜덤과정과는 반대 개념으로 모든 것이 결정되어 있는 과정
    • 시간의 흐름에 따라 일정한 값을 가짐

정상 시계열 데이터

  1. 정상성(stationary)
    • 평균과 분산과 같은 통계적 특성이 시간에 대해 일정한 것을 의미
    • 이러한 특성이 있는 랜덤과정을 정상과정(stationary process) 라고 함
  2. 주식시장
    • 주식 시장과 같은 경우는 비정상과정이지만,
    • 주가 데이터를 분석하는 데 사용하는 방법이 정상성과 관련된 것들이 많음
  3. 약한 정상성
    • weak-sense stationary
    • 평균 함수와 공분산함수가 시간에 대해 변하지 않는 성질을 약한 정상성이라고 하고, 이러한 과정을 약한 정상성 과정이라 함
    • 특성
      • 일정한 평균
      • 일정한 분산
      • 시간에 독립적인 공분산

랜덤과정에서의 기대값, 분산, 공분산

  • 2개의 변수의 상관정도를 나타내는 값
  • 2개 변수의 방향성을 파악하는 용도로 활용
  • 공분산 값의 크기에 따라 상관관계의 강함과 약함을 이야기 할 수 없음
  • 해석 방법
    • 공분산 > 0 : 2개의 변수가 같은 방향으로 움직임
    • 공분산 < 0 : 2개의 변수가 반대 방향으로 움직임
    • 공분산 = 0 : 선형적인 관계가 없으며, 독립적인 관계임

상관

  • 2개의 변수 간에 어떤 선형적인 관계가 있는지를 분석하는 방법
  • 상관계수로 분석
  • 공분산과는 달리 방향성과 함께 상관관계의 정도를 나타냄
  • 해석 방법
    • 상관계수 > 0 : 양의 상관관계
    • 상관계수 = 1 : 2개 변수가 동일
    • 상관계수 < 0 : 음의 상관관계
    • 상관계수 = -1 : 2개의 변수가 음의 방향으로 동일
    • 상관계수 = 0 : 선형적인 상관관계가 없음
  • 상관계수의 절대값이 0.3보다 작으면 약한 관계로, 0.7보다 크면 강한 관계로 보통 해석함

자기공분산

  • 약한 정상성 과정 변수의 공분산 함수
  • 공분산이 동일한 시간에서 2개 변수의 상관관계를 분석하는 것이라면, 자기공분산함수는 서로 다른 2개의 시간에 대한 변수 값의 공분산을 계산한 값
  • 시간에 따른 값들의 상관관계가 어떻게 되는가를 파악하는데 목적이 있음
  • 상관관계가 증가추세인지 하향추세인지, 상관관계의 변화폭은 얼마나 되는지를 살펴볼 수 있음

자기상관

  1. 자기상관이란
    • 시계열 변수의 시간에 따른 자기상관관계를 타나내는 것
    • 계열상관(serial correlation) 또는 교차자기상관(cross correlation)이라고 함
    • 시간의 변화에 따른 변수 간의 상관관계 변화를 관찰하는 것이 목적
    • 자기상관은 자기공분산(k)/자기공분산(0) 으로 정의
    • 자기상관을 활용하면 데이터의 무작위성(randomness)를 파악할 수 있음
    • 자기상관 그래프를 그렸을 때 데이터가 0가 가까울수록 무작위성이 있는 시계열 데이터로 판단할 수 있고, 0보다 큰 값을 가질 때 자기상관을 강하게 가지고 있음
  2. 상관도표
    • 상관도표는 자기상관 함수를 lag K의 순차적인 값에 따라 그린 그래프로 자기상관 그래프와 동일
    • 상관도표를 계절 효과인 계절성이나 결정적인 추세가 있는지를 감지하기 위해 많이 사용

랜덤위크

  1. 랜덤워크란
    • 이전 행보와 독립적인 무작위 행보가 임의의 방향으로 진행되는 것으로 이산변수와 연속변수에 모두 사용할 수 있음
    • 한 지점에서 다음 지점까지의 거리가 일정해 평균은 일정하지만 방향성은 무작위로 결정되어 분산이 시간이 지나감에 따라 증가하는 모습을 보이는 특징을 가짐
    • 종류
      • 표류 없는 랜덤워크(random walk without drift) : 분산이 일정
      • 표류하는 랜덤워크(random walk with drift) : 분산이 시간이 지남에 따라 확대되는 모델
      • 차이는 분산의 변동성에 따라 나눔
  2. 기하적 브라운 운동
    • 랜덤워크의 일종으로 표류 브라운 운동을 하는 랜덤 워크임
      \[ X(t) = z_0 exp(\mu t + \sigma W(t)) \]

chapter 4. 알고리즘 트레이딩

알고리즘 트레이딩 소개

  • 알고리즘 트레이딩이은 수학적 계산과 IT 시스템을 이용해 금융 상품을 거래하는 것
  • 시스템 트레이딩, Algo 트레이딩, blackbox 트레이딩 등으로 불림
  • 수학적 모델을 만드는 과정에서 제일 중요한 핵심은 검증!!!
  • 알고리즘 트레이딩의 장점
    • 속도가 빠름
      • 사람이 대처하는데는 속도의 한계가 있는데 반해, 알고리즘 트레이딩을 활용하면 모델에 따라 빠르게 판단하고 거래할 수 있음
      • 이러한 특성을 극대화한 것이 고빈도 매매(HFT, High Frequency Trading)
    • 감정이 없음
      • 수학적 모델에 기반을 두고 결정하므로 투자의 장애물이 감정이 개입할 여지가 없음
  • 알고리즘 트레이딩의 단점
    • 시장의 교란
      • 너무 많은 거래를 발생시켜 시장의 변화가 빠르고 혼란스러워 짐
      • 우연의 일치 또는 비슷한 알고리즘 트레이딩의 영향으로 인해 시장을 폭락 또는 폭등 시킬 수 있음
    • 프로그램 오류 및 수학적 모델 실패
      • 모델이 잘못 설계 되었거나 구축한 시스템에 오류가 있으면 엄청난 손해를 초래할 수 있음

알고리즘 트레이딩 모델

  • 알고리즘 트레이딩에서의 모델은 주식의 가치를 평가하고 이에 따라 매수/매도/보유를 결정하기 때문에 위험관리, 거래비용관리, 포트폴리오 구성, 검증 등에서 아주 중요한 위치를 차지함
  • 주로 사용되는 모델
    • 평균 회귀(mean reversion)
    • 일간 모멘텀(interday momentum)
    • 추세 추종(trend following)
    • 인덱스 펀드 재분배(index fund rebalancing)
  • 평균 수익보다 월등한 수익을 내는 모델을 알파 모델이라고 하고, 시장 평균이나 이를 약간 웃도는 정도를 베타 모델이라고 함
  • 접근 방법
    • Theory-driven
      • 어떤 모델을 가정하고 그 모델을 검증함으로써 알파 모델을 완성해 나가는 방법
      • top-down 방식
      • 가설을 기반으로 모델을 만들기 때문에 설계자의 측면에서 보면 자신의 모델이 무엇인지, 여러 변수가 어떻게 영향을 미치는지를 잘 이해할 수 있음
    • Data-driven
      • 데이터를 분석해 패턴을 찾고 이를 알파 모델로 만드는 방법으로 뚜렷한 가설이나 이론적 배경없이 데이터를 분석하면서 지식을 쌓아가는 방식
      • bottom-up 방식
      • 가설을 세우지 않고 데이터 분석을 시작하는데, 뚜렷한 목적이 없으면 분석 시에 의미를 찾기가 어렵고 어떤 패턴을 발견해도 이를 이해하기가 어려움
    • 알고리즘 트레이딩을 하는 사람들은 주로 theory-driven 방식을많이 사용하는데, 이가 훨씬 익숙하고 모델에 대한 이해도가 높다는 점 때문에 이렇게 많이들 함

평균회귀 모델

  1. 기본 가정
    • 시계열 데이터는 과거 평균값으로 회귀하려는 경향이 있다
    • 어떤 변수가 정규분포를 따른다고 하면, 평균에 가까이 갈 확률이 높아지고 멀어질수록 확률이 낮아지는 점을 활용
    • 현시점의 주가가 평균 주가보다 낮으면 주가가 올라가려고 하고, 반대로 현시점의 주가가 평균 주가보다 높으면 주가가 내려가려 하는 경향을 이용한 것
  2. 평균회귀 테스트
    • 평균회귀 모델을 적용하려면 적용하려는 주가 데이터가 랜덤워크인지 아닌지를 파악해야 함
    • 랜덤워크라면 평균회귀 모델을 적용할 수 없음
    • ADF(augmented dickey-fuller) 테스트
      • 어떤 주가가 평균회귀 성질을 가지고 있다면 현주가는 다음의 주가가 어떻게 될지 예측할 수 있는 통계적 정보를 포함하고 있음
      • 어떤 주식의 주가가 랜덤워크를 따른다는 가정을 세우고, 이 가설을 검증해 랜덤워크인지 판단하는 방법
